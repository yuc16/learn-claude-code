# s02 总结

本章介绍如何在不修改智能体主循环的前提下扩展工具能力。核心思想是把工具调用从单一的 `bash` 扩展为一个“工具分发器”：为每个工具实现独立 handler，再用 `TOOL_HANDLERS` 这样的映射表按工具名分发执行。这样新增 `read_file`、`write_file`、`edit_file` 等工具时，只需要补充 schema 和 handler，而不用改 agent loop。

文中还强调了专用工具相比 bash 的优势：它们更稳定、更容易约束，并且可以通过 `safe_path()` 把文件访问限制在工作区内，降低路径逃逸和误操作风险。总体上，s02 的重点不是改变循环，而是通过“分发 map + 路径沙箱”让智能体安全、清晰、可扩展地使用多种工具。
