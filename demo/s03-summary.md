# s03 总结

本章引入 `TodoWrite` / `todo` 机制，解决智能体在多步任务中容易丢进度、重复操作和执行跑偏的问题。核心做法是为智能体增加一个带状态的待办列表，让它先拆解步骤，再按顺序推进。

`TodoManager` 负责保存任务项，并强制**同一时间只能有一个任务处于 `in_progress`**，从而让模型聚焦当前步骤，减少并行发散。与此同时，系统还加入了 nag reminder：如果模型连续 3 轮没有更新 todo，就会自动注入提醒，督促它回到计划管理上。

相较前一章，这一节不是改变基础循环，而是在原有 agent loop 上增加规划与状态约束。结果是智能体在处理长链路、多步骤任务时更稳定，也更不容易忘记已经完成或正在处理的工作。
